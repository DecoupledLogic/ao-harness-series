# Release Notes

Per-release notes for the AgenticOps Harness series. One file per tag, named `<tag>.md` (e.g. `v0.1.0.md`).

Release notes are evidence. They let anyone look at a tag and see exactly which slice it bundles, what acceptance criteria it met, and the build and test result captured at release time, without reading the commit history. Each file lists the slice that shipped, links its spec, and records the verification that gated the tag.
